\begin{frame}
    \frametitle{Tipos de dados em Python}
    \begin{itemize}
        \item Similarmente à Matemática, em Python e noutras linguagens de
            Programação, existem conjuntos de valores, denominados tipos de dados.
        \item No entanto, estes não se limitam apenas a números, existem também 
            formas de representar texto, verdadeiros ou falsos, entre outros.
        \item Um valor pode ter diferentes representações, mas cada representação
            apenas está associada a um tipo de dados. Por exemplo, 1 é um valor
            inteiro, enquanto 1.0 é considerado real.
    \end{itemize}
\end{frame}

\begin{frame}
    \frametitle{Algumas notas sobre Floats}
    \begin{itemize}
        \item Não é possível representar fidedignamente todos os números 
        reais, como por exemplo, os números com dízimas infinitas, já que os 
        computadores têm espaço de armazenamento finito.
        \item Outro exemplo clássico é somar \textit{0.1} com \textit{0.2}, que 
        resulta em \textit{0.30000000000000004}.
        \item Isto deve-se à norma IEE 754, usada para representar números reais,
        mas que não será discutida nestes materiais.
    \end{itemize}
\end{frame}

\begin{frame}
    \frametitle{Expressões aritméticas}
        Um dos usos mais simples das linguagens de Programação é como
        calculadora, através dos seguintes operadores aritméticos básicos: 
        \begin{itemize}
            \item $+$ representa a soma. Ex:
            \begin{itemize}
                \item $2+2$, que é igual a $4$
                \item $-1.2+3.6$, que é igual a $2.4$
            \end{itemize}
            \item $-$ representa a subtração. Ex:
            \begin{itemize}
                \item $3-1$, que é igual a $2$
                \item $5.3-1.8$, que é igual a $3.5$
            \end{itemize}
            \item $*$ representa a multiplicação. Ex:
            \begin{itemize}
                \item $5*2$, que é igual a $10$
                \item $2.9*3$, que é igual a $8.7$
            \end{itemize}
        \end{itemize}
\end{frame}

\begin{frame}
    \frametitle{Expressões aritméticas}
        \begin{itemize}
            \item $/$ representa a divisão real. Ex:
            \begin{itemize}
                \item $4/2$, que é igual a $2.0$
                \item $5.4/0.25$, que é igual a $21.6$
            \end{itemize}
            \item $//$ representa a divisão inteira. Ex:
            \begin{itemize}
                \item $4//2$, que é igual a $2$
                \item $5//2$, que é igual a $2$
            \end{itemize}
            \item $\%$ representa o resto da divisão inteira. Ex:
            \begin{itemize}
                \item $4//2$, que é igual a $0$
                \item $5//2$, que é igual a $1$
            \end{itemize}
            \item $**$ representa a potência. Ex:
            \begin{itemize}
                \item $2**4$, que é igual a $16$
                \item $9**0.5$, que é igual a $3.0$
            \end{itemize}
        \end{itemize}
\end{frame}

\begin{frame}
    \frametitle{Notas sobre expressões aritméticas}
        \begin{itemize}
            \item Tal como na Matemática, a divisão por $0$ não está definida em
                Python e caso seja efetuada irá produzir um \textbf{erro}.
            \item A divisão real produz obrigatoriamente um valor do tipo \textbf{float}
            \item Tanto o resto como a divisão inteira produzem valores do tipo
            \textbf{int}
            \item O tipo de dados do resultado da potência \textbf{depende} do 
            tipo de dados da base e do exponente, se pelo menos um deles for 
            float o resultado será float, se ambos forem inteiros será inteiro.
    
        \end{itemize}
\end{frame}

\begin{frame}[containsverbatim]
    \frametitle{Exercício}
    Escreva um programa em Python que calcule a distância eucliadiana entre 
    os pontos (3, 4) e (7, 1). A fórmula para dois pontos ($x_1$, $y_1$)
    e ($x_2$, $y_2$) é:
    \vspace{2mm}
    \[ d = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2} \]
    \vspace{4mm}
    Talvez a solução mais intuitiva seja:
    \begin{pythonlst}
        import math

        print(math.sqrt((3 - 7)**2 + (4 - 1)**2))
    \end{pythonlst}
\end{frame}

\begin{frame}
    \frametitle{Repetição de código}
    E se nos pedirem para calcular a distância entre outros dois pontos?
    Simples, fazer copy paste!\\
    \vspace{4mm}
    No entanto, isto introduz alguns problemas:
    \begin{itemize}
        \item A fórmula da distância é relativamente simples, mas copiar pedaços 
            de código mais complexos e com múltiplas linhas torna o código 
            difícil de ler.
        \item Se detetarmos um erro numa fórmula seria necessário substituir em 
            vários locais, o que não é desejável, e inclusivamente ficar a faltar
            em algum sítio.
    \end{itemize}
    Qual será a melhor solução?
\end{frame}

\begin{frame}[containsverbatim]
    \frametitle{Funções}
    Uma solução mais adequada seria:
    \begin{pythonlst}
        import math

        def dist(x1, y1, x2, y2):
            return math.sqrt((3 - 7)**2 + (4 - 1)**2)
        
        print(dist(3, 4, 7, 1))
        print(dist(0, 0, 0, 1))
    \end{pythonlst}
\end{frame}

\begin{frame}[containsverbatim]
    \frametitle{Funções}
    No entanto, a melhor solução seria:
    \begin{pythonlst}
        import math

        def dist(x1, y1, x2, y2: float) -> float:
            """ Calcula a distancia eucliadiana entre dois pontos
                x1, y1 - Coordenadas do ponto 1
                x2, y2 - Coordenadas do ponto 2
                Retorna a distancia como numero real """
            return math.sqrt((3 - 7)**2 + (4 - 1)**2)
        
        print(dist(3, 4, 7, 1))
        print(dist(0, 0, 0, 1))
    \end{pythonlst}
\end{frame}

\begin{frame}
    \frametitle{Condicionais}
    Em Python é possível expressar alternativas através das keywords $if$, $elif$
    e $else$. Um bloco condional representa um conjunto de alternativas associadas
    entre si. Cada bloco começa obrigatoriamente com um único $if$, no entanto
    é possível omitir tudo o resto (isto representa não fazer nada se a condição
    original for falsa). Cada bloco condional pode ter $0$ ou mais $elif$'s, 
    consoante o número de casos que queiramos distinguir. Já o $else$ representa 
    o caso em que todas as outras condições verificadas anteriormente são falsas,
    ou seja, por bloco condional apenas pode existir no máximo um $else$ e deverá
    ser a última expressão desse bloco condional, pois engloba todos os restantes
    casos.
\end{frame}
